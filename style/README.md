# style/ — стиль письма автора (frontstage)

Единый дом авторских **контент-стилей**: ПРАВИЛА «как автор звучит» в публикуемом контенте. Отделён от самого контента (тексты): посты в `docs/`, видео-сценарии в `video/themes/`. Правило ≠ контент.

| Файл | Канал | Что это |
|------|-------|---------|
| `post.md` | посты, эссе, лонгрид | Канон стиля постов. Полное выражение голоса автора (вынесен из CLAUDE.md §9/§9a). |
| `video.md` | короткие ролики (45-75 сек) | Стиль видеоречи. Тот же голос, override под устный короткий формат. Наследует `post.md`. |

## Архитектура (тир)

- **Платформенный КАРКАС** стилей (регистры, движок, различения голос/тон и поле/отношение/канал) — в `PACK-rhetoric/language-style`, общий для всех пользователей IWE.
- **Эти файлы** — ЛИЧНЫЙ контент-голос автора (frontstage). Реестр PACK-rhetoric **ссылается** на них (`content-sources.yaml`: `L1.post-style`, `L1.video-speech`), не копирует. Личный голос в платформенный пак не уходит (иначе уехал бы всем пользователям).
- **Служебный голос агента** (backstage: как агент пишет пилоту и код) — отдельно, в `DS-my-strategy/memory` (`communication-style-author.md`, `code-style-author.md`).

Две семьи голоса по аудитории: backstage (служебный) ≠ frontstage (публичный, эта папка). Над обеими — тир: каркас платформенный, голос личный.

Источник решения: peer-сессия `DS-my-strategy/sessions/2026-06/2026-06-20-15-style-canon-consolidation/`, WP-335 Ф8.
