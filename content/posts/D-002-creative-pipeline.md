# Творческий конвейер: от заметки до поста без застревания

Знакомая ситуация: заметки копятся, черновики начаты и брошены, посты «почти готовы» неделями. Накопление без движения — главный враг творческого конвейера. Решение: каждый артефакт имеет срок жизни и обязательный выход. Ничего не застревает.

## Четыре стадии артефакта

Каждая мысль проходит четыре стадии. Ключевое: это не четыре папки, а четыре *состояния* с разными правилами.

| Стадия | Что это | Кто видит | Где хранится | TTL |
|--------|---------|-----------|-------------|-----|
| **Заметка** | Мгновенная мысль, наблюдение, цитата | Только я | fleeting-notes.md | 7 дней |
| **Черновик** | Связный текст из нескольких заметок | Только я + ИИ-ассистент | drafts/ (приватный репо) | 7 дней |
| **Заготовка** | Текст, понятный единомышленникам | Сообщество (публичный git) | DS-Knowledge-Index, status: draft | 14 дней |
| **Пост** | Оформленный текст для аудитории | Все (клуб) | DS-Knowledge-Index, status: published | — |

### Различение: черновик и заготовка — это не одно и то же

В методе Thinking in Writing (PD.METHOD.004) оба называются «draft». Но на практике это два разных артефакта:

- **Черновик** — личный, сырой, для себя. Можно писать плохо, перескакивать, не структурировать. Тест: «Покажу ли я это кому-то?» — Нет.
- **Заготовка** — публичная, для единомышленников. Требует минимальной структуры: понятно, о чём и зачем. Тест: «Поймёт ли коллега из клуба?» — Да.

Зачем разделять? Потому что смешение приводит к двум типичным ошибкам:
1. **Перфекционизм** (PD.FAIL.014): применяем стандарт поста к тому, что должно быть черновиком — не можем начать.
2. **Преждевременная публикация**: публикуем сырые мысли вместо связного текста — теряем доверие читателя.

## Правило незастревания (TTL)

Каждый артефакт имеет **срок жизни** — TTL (time-to-live). Если за TTL артефакт не продвинулся, на ближайшем стратегировании принимается решение:

1. **Приоритизировать** — вынести в топ-7 и обработать на неделе.
2. **Продвинуть** — превратить в следующую стадию (черновик → заготовка).
3. **Закрыть** — осознанно отправить в архив с причиной. Не удалять молча.

Конкретные TTL:

| Стадия | TTL | Кто проверяет |
|--------|-----|---------------|
| Заметка | 7 дней | Note-Review (ежедневно, 23:00) |
| Черновик | 7 дней | Strategy Session (еженедельно) |
| Заготовка | 14 дней | Strategy Session (еженедельно) |

**Тест работающего конвейера:**
- В fleeting-notes.md нет заметок старше 7 дней? ✅
- В drafts/ нет черновиков старше 7 дней? ✅
- В DS-Knowledge-Index нет заготовок (status: draft) старше 14 дней? ✅

Если хотя бы один тест не пройден — конвейер застрял.

## Замкнутый цикл

Конвейер работает не линейно (заметка → пост), а **циклически**. Три обратных связи замыкают его:

### 1. Публикация → Pack (фиксация знания)

При написании поста часто кристаллизуется новое знание: различение, метод, типичная ошибка. Это знание нужно зафиксировать в Pack — иначе оно существует только в посте и не переиспользуется.

```
Пост → обратная связь → captures.md → KE → Pack
```

### 2. Pack → черновик (популяризация)

Когда Pack обогащается новыми сущностями (≥3 по одной теме), это триггер для нового поста. Pack содержит формализованное знание, но его нужно *перевести* на язык аудитории.

```
Pack обогащён → draft-list.md → черновик → заготовка → пост
```

### 3. Обратная связь → заметка (ревизия)

Комментарии к посту — источник новых заметок. Кто-то указал на ошибку? Предложил другой взгляд? Это возвращается в fleeting-notes и запускает цикл заново.

```
Комментарий → fleeting-notes.md → Note-Review → ...
```

## Полная схема

```
┌──────────────────────────────────────────────────────┐
│                                                      │
│  Изучение / чтение / наблюдение                      │
│              ↓                                       │
│     ЗАМЕТКА (fleeting-notes, TTL 7д)                 │
│              ↓                                       │
│     Note-Review (ежедневно) — маршрутизация:         │
│       ↓              ↓              ↓                │
│  captures.md    ЧЕРНОВИК (7д)    РП в WeekPlan       │
│  «→ Pack»      drafts/           «→ Задача»         │
│       ↓              ↓                               │
│  KE → Pack      ЗАГОТОВКА (14д)                      │
│       ↓         DS-Knowledge-Index                   │
│       ↓         status: draft                        │
│       ↓              ↓                               │
│       ↓         ПОСТ (status: published)             │
│       ↓              ↓                               │
│  Pack обогащён ← обратная связь                      │
│       ↓                                              │
│  Триггер: ≥3 сущностей по теме → новый черновик     │
│       ↓                                              │
│  draft-list.md ─────────────────────────────→────────┘
```

## Реализация в экзокортексе

Конвейер реализуется тремя инструментами:

1. **draft-list.md** — индекс всех черновиков с приоритетами (в DS-strategy/current/).
2. **drafts/** — папка с файлами черновиков (в DS-strategy/).
3. **Note-Review** — ежедневная сортировка заметок с маршрутизацией в черновики.

На стратегировании (раз в неделю):
- Ревью draft-list.md — что застряло?
- TTL-проверка — есть ли просроченные?
- Приоритизация — какие 3-7 черновиков продвигаем?
- Триггер Pack → пост — есть ли новое знание для популяризации?

## Чего НЕ должно быть

- **Заметки старше 7 дней** — значит Note-Review не работает.
- **Больше 10 черновиков** — значит не приоритизируем, а копим.
- **Заготовки старше 14 дней** — значит пишем, но не публикуем.
- **Посты без captures** — значит знание не возвращается в Pack.
- **Pack без постов** — значит знание формализовано, но не популяризовано.

Творческий конвейер работает только когда он замкнут и ничего не застревает. TTL — механизм, который делает это проверяемым.
