# Модульный IWE: как перестать выдавать всё сразу

> Черновик поста для клуба. РП #96.

---

Шаблон IWE (Intellectual Work Environment) — это персональная рабочая среда с ИИ-агентами. Вы устанавливаете шаблон, и Claude Code начинает помогать: планирует день, фиксирует знания, ведёт учёт задач.

Сейчас шаблон работает. Но есть проблема, которую я хочу обсудить.

---

## Что есть сейчас

После установки шаблона пользователь получает:

- **CLAUDE.md** — системные инструкции для Claude Code (~150 строк правил)
- **Экзокортекс** (memory/) — 8 файлов: протоколы сессий (ОРЗ), терминология, чеклисты, роли
- **3 ИИ-роли** — Стратег (утренний план, недельный обзор), Экстрактор (фиксация знаний в Pack), Синхронизатор (расписание, уведомления)
- **DS-strategy/** — репозиторий для планов, задач, архивов
- **Автоматизация** — launchd-агенты (macOS), cron (Linux), Telegram-уведомления
- **MCP-серверы** — поиск по базе знаний (5400 документов), цифровой двойник
- **WakaTime** — трекинг времени
- **Интеграция с ботом** (@aist_me_bot) — ежедневная поддержка через Telegram

Это ~50 файлов: скрипты, промпты, конфиги, seed-данные. Установка через `setup.sh` с заменой 8 переменных.

---

## В чём проблема

Я получил обратную связь от нескольких потенциальных пользователей. Вот типичные ситуации:

**«Мне нужно просто вести свой проект.»** Человек хочет протокол ОРЗ (Открытие → Работа → Закрытие) и фиксацию знаний. Ему не нужны бот, MCP, автоматизация, launchd. Но шаблон выдаёт всё сразу — и человек теряется в сложности, которая ему не нужна.

**«У меня уже есть проекты и данные.»** Заметки в Notion, код на GitHub, документы в папках. Как это всё перенести в IWE? Шаблон не предлагает пути миграции — он предполагает, что вы начинаете с нуля.

**«Слишком много зависимостей.»** Бот, MCP-серверы, WakaTime, GitHub CLI, Node.js — каждая зависимость это решение «доверять или нет», «настраивать или нет», «что сломается если обновится». Для человека, который хочет просто начать — это барьер.

Суть: **шаблон монолитный, а пользователи разные**.

## Предложение: Core + Modules

Идея: разделить шаблон на **ядро** (минимально необходимое) и **модули** (добавляются по желанию).

### Ядро (Core)

Самодостаточный минимум. Зависимости: `git` + любой AI CLI (Claude Code, Codex, Aider, Cursor).

Что входит:
- **CLAUDE.md** (~50 строк) — ОРЗ + Capture-to-Pack + базовые правила
- **memory/** (4 файла) — упрощённые протоколы Open/Work/Close + MEMORY.md
- **DS-strategy/** — планы, задачи, архив

Что НЕ входит: никаких ролей, скриптов, MCP, бота, launchd, WakaTime.

Установка: `bash setup.sh --lite` → 4 файла, 5 минут, работает.

### Модули

Каждый модуль — папка с `install.sh`, который:
1. Добавляет секцию в CLAUDE.md
2. Копирует нужные memory-файлы
3. Настраивает автоматизацию (если есть)

Предварительный список модулей:

| Модуль | Что добавляет | Зависимости |
|--------|-------------|-------------|
| **wp-tracking** | WP Gate (реестр задач, контексты, нумерация) | — |
| **strategist** | Роль Стратег (утренний план, недельный обзор) | Claude Code + launchd/cron |
| **extractor** | Роль Экстрактор (фиксация знаний в Pack) | Claude Code |
| **synchronizer** | Роль Синхронизатор (расписание, уведомления) | bash + cron |
| **mcp-knowledge** | Поиск по базе знаний | MCP-сервер |
| **mcp-ddt** | Цифровой двойник | MCP-сервер + Neon DB |
| **archgate** | АрхГейт (оценка архитектурных решений) | — |
| **import** | Миграция существующих проектов и данных | — |

### Профили (для быстрого старта)

Чтобы не выбирать модули по одному — три готовых профиля:

- **Lite** = Core. Для тех, кому нужен только проект + ОРЗ.
- **Standard** = Core + wp-tracking + strategist. Планирование и трекинг, без внешних сервисов.
- **Full** = Core + все модули. Текущее поведение шаблона.

```
bash setup.sh --lite       # 5 минут, 4 файла
bash setup.sh --standard   # 10 минут, планирование
bash setup.sh              # 15 минут, всё (текущее)
```

## Импорт существующих данных

Отдельный модуль `import/` с тремя сценариями:

**Git-репозитории:** Скрипт спрашивает путь и тип (Pack или DS?), перемещает в рабочую папку, добавляет минимальный CLAUDE.md.

**Неструктурированные файлы:** Claude Code помогает рассортировать — стратегические заметки в DS-strategy/inbox, доменные знания предлагает оформить как Pack.

**Данные из других систем:** Экспорт из Notion/Obsidian → конвертация → размещение по правилам IWE.

## Что это меняет для пользователя

| Сейчас | После |
|--------|-------|
| «Установи всё, потом разберёшься» | «Начни с минимума, добавляй по мере надобности» |
| 50 файлов сразу | 4 файла (Lite) → 15 (Standard) → 50 (Full) |
| Бот + MCP обязательны | Бот и MCP — отдельные модули |
| Нет пути миграции | Модуль import для существующих данных |
| Один размер для всех | Три профиля под разные потребности |

## Вопросы для обсуждения

1. **Какой профиль вам ближе?** Lite (только проект + ОРЗ), Standard (+ планирование), Full (всё)?

2. **Что должно быть в ядре?** Сейчас предлагаю минимум: CLAUDE.md + 3 протокола + DS-strategy. Может, что-то ещё обязательно нужно с первого дня?

3. **Какие модули не хватает?** Например: интеграция с Obsidian, шаблоны постов, учебные материалы?

4. **Импорт данных — актуально?** У вас есть существующие проекты/заметки, которые хотели бы перенести в IWE?

5. **Что пугает в текущем шаблоне?** Что стало бы барьером, если бы вы решили попробовать IWE?
